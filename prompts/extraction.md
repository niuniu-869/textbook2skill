# 章节抽取 prompt 模板

`extract.py` 会读这个文件，提取代码块内的内容作为 prompt。`{chapter_text}` 会被替换成章节原文。

```
你正在为 Claude Code 制作一个**领域知识 skill 片段**。读这份输出的不是学生，是另一个 LLM——它需要在 1-2 秒内消化你的内容并准确回答用户问题。

输入是教材某一章的原文 OCR 结果，可能有少量识别错误。你的任务是把整章压缩成一份**密度极高、零废话**的 markdown 文件。

---

## 输出结构（严格遵守，不要加前言/总结/解释）

\```markdown
# {完整章节号 + 标题，如"第3章 本量利分析"}

## 核心概念
- **{术语}**: {精确定义，含计量单位/范围限定}
- ...
（4-10 个，按出现顺序排列）

## 公式与计算口径
- **{公式名}**: `公式表达式` — {各符号含义 + 适用前提}
- ...
（如果本章无公式，写"本章不涉及定量公式"。否则全部列出）

## 方法/流程
1. {可执行的步骤，含判断条件}
2. ...
（仅在本章涉及"怎么做"时填写。纯概念章节写"无标准化流程"）

## 例题
### 例 N.M: {简短问题陈述}
- **已知**: {完整数据}
- **求**: {目标}
- **解**:
  1. {步骤} → 结果
  2. ...
- **结论**: {最终答案}

（保留 1-3 个**最有代表性**的例题。挑判断标准：①涵盖本章核心方法 ②含完整数字 ③解题路径不止一步。不符合的例题不收录）

## 易混点 / 边界条件
- {只在原书明确强调或反复对比时收录}
（如无，本节省略整段）

## 关联
- 前置: {依赖的前置概念/章节}
- 应用: {后续如何被用到}
（不知道写"暂无明确说明"）
\```

---

## 硬性要求

1. **中文输出，markdown 格式**，不要写 "好的、我来帮你..."、"以下是..." 等开场白，第一行直接是 `#`
2. **术语保留原貌** — 不要翻译、同义改写、口语化（"企业使命" 不要简化为 "使命"）
3. **数字与公式 100% 准确** — 宁可省略也不要乱编。OCR 错乱的地方写 `[OCR错乱]`
4. **杜绝填充语** — 不要写"首先...其次...最后"、"非常重要"、"需要注意"。Claude 不需要被提醒"重要"
5. **每个 bullet ≤ 30 字** — 如果一句话讲不完，拆成多个 bullet
6. **例题里的解题步骤必须含具体数字** — 不写"按公式计算" 而写"5000 × 0.6 = 3000"

---

## 反例（不要这样输出）

❌ "本章主要介绍了本量利分析的相关概念，包括..."  ← 废话开场
❌ "**贡献边际**: 一个非常重要的概念..."  ← 没定义
❌ "公式: 销售额减去变动成本"  ← 没保留原书表达式
❌ "例题: 某企业生产 A 产品..."  ← 缺已知/求/解结构

---

```

## 调用要点

- **绝不传 max_tokens / temperature**（reasoning 模型陷阱）
- 11 章并发跑，`ThreadPoolExecutor(max_workers=11)`
- 单章 LLM 失败时 `extract.py` 已捕获，写占位文件 `[抽取失败]`

## 验证抽取质量

输出后人眼/Claude 检查：
- 第一行是 `#` 开头，不是 "好的" / "以下是"
- `## 核心概念` 至少 4 条，`**术语**: 定义` 格式
- `## 公式` 数量 ≥ 章节实际公式 70%
- `## 例题` 含具体数字
- 总长度：原章节字符 / 输出字符 = 5x ~ 10x（压缩比合理）

如果不达标 → 调 prompt 后重试该章节。
